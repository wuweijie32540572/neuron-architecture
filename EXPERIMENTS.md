## v4: 两阶段训练验证

### 实验设置

- **任务**: 正弦波时序预测
- **训练数据**: 200样本
- **测试数据**: 100样本
- **架构**: SCH(16) + PC(16)

### 结果

```
预训练:
  初始loss: 0.284
  最终loss: 0.004
  改进率: 98.7%

测试:
  预训练后MSE: 0.081
  在线适应后MSE: 0.008
  改进: +90.7%

异常检测:
  正常样本平均分: 0.019
  异常样本平均分: 0.032
  区分度比值: 1.66x
```

### 对比实验

```
v3 纯局部学习 MSE: 6.64
v4 两阶段训练 MSE: 0.12
改进: 98.1%
```

### 关键发现

**结构性矛盾的解决方案**:

局部学习信号 ε=z-μ 与任务目标 (x-x̂) 之间存在结构性断裂。两阶段训练通过以下方式解决：

1. **离线预训练**: 用任务监督信号建立 z→x̂ 的正确映射
2. **在线适应**: 用局部学习微调，保持可塑性

## v5.1: HM记忆层验证

### 实验设置

- **任务数**: 8个真正正交的冲突任务
- **任务类型**: sin、线性、指数、阶跃、绝对值、二次、符号、对数
- **每任务样本**: 100
- **架构**: SCH(8) + PC(8) + HM(4)

### 任务定义

```
任务0: y = sin(2πx)           # 周期
任务1: y = 2x - 1             # 线性
任务2: y = exp(x) - 1.5       # 指数
任务3: y = step(x - 0.5)      # 阶跃
任务4: y = |2x - 1| - 0.5     # 绝对值
任务5: y = 4(x - 0.5)² - 0.5  # 二次
任务6: y = sign(x - 0.5)      # 符号
任务7: y = log(x + 0.1) / 2   # 对数
```

### 结果

```
遗忘率:
  带HM: -11.6% (性能改善)
  无HM: +9.2%
  HM缓解效果: 20.8个百分点

任务0性能变化:
  阶段1: MSE=0.6373 (学任务0后)
  阶段4: MSE=1.0189 (学任务3后，最差)
  阶段8: MSE=0.4274 (学任务7后)
```

### 关键发现

**记忆稳定性度量的有效性**:

```
σ(m) = 1 / (1 + α·age + β/(access+1))
```

- 高age的记忆稳定性低，被优先重放
- 高access的记忆稳定性高，被保护

**风险加权重放的效果**:

重放时按 risk(m) ∝ 1-σ(m) 加权采样，优先巩固高风险记忆。

## v6.2: DA机制验证

### 实验设置

- **DA水平**: 高(0.9)、中(0.5)、低(0.1)
- **训练步数**: 200
- **目标输出**: 0.8

### 学习率衰减曲线

```
步数    高DA(0.9)  中DA(0.5)  低DA(0.1)
0       0.0725     0.0625     0.0525
100     0.0659     0.0417     0.0276
500     0.0483     0.0179     0.0095
```

### 结果

```
收敛分析:
  高DA: 最终误差=0.735, 收敛速度=1.15x
  中DA: 最终误差=0.689, 收敛速度=1.08x
  低DA: 最终误差=0.672, 收敛速度=1.05x

相对优势:
  初期高DA优势: +0.9%
  后期低DA优势: +3.0%
```

### 关键发现

**DA控制探索/利用权衡**:

- 高DA: 学习率衰减慢，保持探索能力
- 低DA: 学习率衰减快，快速稳定

这符合生物多巴胺的功能：高DA表示奖励预测误差，驱动探索；低DA表示预期满足，转向利用。

  遗忘率(最差): +194.8% ✗

异常检测:
  平均检测比: 2.53x ✓
```

### 任务0详细性能

```
阶段1 (学sin):      MSE=0.5555
阶段2 (学线性):     MSE=0.7786
阶段3 (学指数):     MSE=0.4859
阶段4 (学阶跃):     MSE=0.4856
阶段5 (学绝对值):   MSE=0.3829  ← 意外改善

### 关键发现

**HM在真正正交任务上的局限性**:

1. 二次函数（任务5）的权重更新与sin函数（任务0）产生剧烈冲突
2. HM重放80个样本不足以抵抗这种冲突
3. 简单的模式分离+重放机制在真正正交任务上效果有限

**异常检测的有效性**:

改为"目标值+随机噪声"的异常定义后，平均检测比达到2.53x，对所有任务类型都有效。

## 真实模型验证 (2M参数 Sequential MNIST)

### 实验设置

- **模型规模**: ~800K参数 (MLP: 512-512-256)
- **数据集**: MNIST (真实手写数字)
- **任务划分**:
  - 任务1: 数字 0-4 (30,592 训练样本)
  - 任务2: 数字 5-9 (29,408 训练样本)
- **训练**: 每任务5 epochs
- **对比方法**: Vanilla、Experience Replay、HM-Neuron

### 结果

```
方法                        任务1初始    任务1最终    遗忘率      缓解效果
-------------------------------------------------------------------------
Vanilla MLP               0.9947       0.0000       99.5%       -
Experience Replay         0.9947       0.9543       4.0%        +95.4pp
HM-Neuron                 0.9979       0.9272       7.1%        +92.4pp
```

### 关键发现

**经验重放极其有效**:

在真实MNIST数据上，经验重放将灾难性遗忘从99.5%降到4.0%，缓解效果达95.4个百分点。

**HM-Neuron同样有效**:

HM-Neuron配合记忆重放，将遗忘率降到7.1%，缓解效果92.4个百分点。

**与合成任务的对比**:

| 任务类型 | 遗忘率(Vanilla) | 遗忘率(Replay) | 缓解效果 |
|----------|-----------------|----------------|----------|
| 合成正交任务 | +194.8% | - | HM不足 |
| 真实MNIST | 99.5% | 4.0% | +95.4pp |

真实任务上效果更好的原因：
1. MNIST数字并非完全正交，存在特征共享
2. 经验重放提供了真实的旧任务数据
3. MLP架构比SCH+PC更适合图像分类

### 运行实验

```bash
python experiments/sequential_mnist_2m_v2.py
```

结果保存到 `results/sequential_mnist_2m_v2.json`
