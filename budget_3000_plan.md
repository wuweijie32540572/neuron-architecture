# 3000元预算方案：从"能跑"到"好用"

## 与2000元方案的核心差异

2000元方案解决了"Python能碰到GPU"的问题（0.14%→100%）。
3000元解决的是：**从"能跑7B"到"能跑35B MoE"**。

多出的1000元，关键投入方向是：**更好的平台 + 更大内存 + 更快存储**。

## 深度推理：1000元增量的最优分配

### 推理1：32GB内存是质变，不是量变

16GB → 32GB 看似翻倍，但实际效果是**跨越了35B MoE模型的门槛**。

35B MoE模型（Q4量化）的内存需求：
- GPU显存：~7GB
- CPU内存：~16GB（MoE专家层offload）
- 系统开销：~4GB
- **总计**：~27GB

16GB内存：❌ 无法运行35B MoE
32GB内存：✅ 可以运行，还有余量

**1000元增量中，230元用于 16GB→32GB 内存，这是最关键的投入。**

### 推理2：NVMe SSD对AI工作流的加速

| 操作 | SATA SSD | NVMe SSD | 加速比 |
|------|----------|----------|--------|
| 加载7B模型(INT4, ~4GB) | ~8s | ~1.5s | 5.3x |
| 加载35B模型(Q4, ~20GB) | ~40s | ~7s | 5.7x |
| 数据集预处理 | 基准 | 3-5x | 显著 |

模型加载虽然是一次性开销，但在频繁切换模型的实验中影响很大。

### 推理3：CPU单核性能对MoE offload的影响

MoE CPU offload的速度取决于CPU单核性能（每个token的专家计算是串行的）：

| CPU | 单核(Cinebench) | MoE offload速度 |
|-----|-----------------|-----------------|
| E5-2666 v3 | ~90 | 基准 |
| R7-3700X | ~200 | ~2.2x |
| i5-12400F | ~240 | ~2.7x |

**1000元增量中，250元用于 E5→R7-3700X 或 i5-12400F，MoE推理速度提升2-3倍。**

## 手机+台式机协同架构（升级版）

```
[Redmi K80 Pro]                    [RTX 3060 12GB 台式机]
    |                                      |
    ├── 传感器数据采集 ──→ WiFi ──→        ├── 训练 Thermo-SNN (CUDA)
    ├── 轻量推理 (NPU/NNAPI)               ├── 跑 35B MoE LLM (~30 tok/s)
    ├── 温度/电池监控                       ├── 跑 7B LLM (~45 tok/s)
    ├── 模型量化/导出                       ├── SNN 实时仿真 (>100Hz)
    |                                      ├── 主动推理模型训练
    └──────── USB-C 直连 ──────────────────┘
                    |
            模型同步/任务分发/传感器数据流
```

**关键协同**：
1. 手机采集传感器数据 → WiFi传到台式机 → 训练主动推理模型
2. 台式机训练好的模型 → 量化为INT8 → 部署到手机NPU
3. 形成闭环：**采集→训练→部署→采集**
